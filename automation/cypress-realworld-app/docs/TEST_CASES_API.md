В документе представлены тест-кейсы для API-тестирования приложения Cypress Real World App.

---

## Содержание
 - TC модуля Авторизация (AUTH) = 8 шт.
 - ТС модуля Личный кабинет = 11 шт.
 - ТС модуля Банковские счета (BANK) = 13 шт.
 - ТС модуля Уведомления (NOTIFICATION) = 1 шт.
 - ТС модуля Создание запросов на денежный перевод (REQUEST) = 11 шт.
 - ТС Полные сценарии по созданию и подтверждению запросов на денежный перевод (REQUEST. E2E) = 1 шт.

# 📝 Тест-кейсы - Уведомления (API)

## TC API NOTIFICATION-1. Пометка уведомления как прочитанного

**Метод:** `GET`, `PATCH`

**Эндпоинты:**

* `GET /notifications`
* `PATCH /notifications/{id}`

**Предусловия**

* Пользователь успешно авторизован.
* У пользователя имеется хотя бы одно непрочитанное уведомление.

**Шаги**

1. Выполнить `GET /notifications`.
2. Получить идентификатор первого непрочитанного уведомления.
3. Отправить `PATCH /notifications/{id}`, передав `isRead = true`.
4. Выполнить повторный `GET /notifications`.
5. Проверить список непрочитанных уведомлений.

**Ожидаемый результат**

* Первый запрос `GET` возвращает HTTP **200**.
* `PATCH` возвращает HTTP **204**.
* Повторный `GET` возвращает HTTP **200**.
* Уведомление, помеченное как прочитанное, отсутствует в списке непрочитанных уведомлений.

## ТС API REQUEST E2E-1. Подтверждение запроса на перевод и изменение балансов пользователей

**Модуль:** Запросы на денежные переводы (API)

### Предусловия

- В системе существуют два пользователя.
- Оба пользователя могут авторизоваться.

### Шаги

1. Авторизоваться под первым пользователем.
2. Создать запрос на перевод.
3. Авторизоваться под вторым пользователем.
4. Получить список транзакций.
5. Найти созданный запрос.
6. Подтвердить запрос.
7. Проверить статус запроса.
8. Проверить баланс второго пользователя.
9. Проверить баланс первого пользователя.

### Ожидаемый результат

- Запрос успешно подтвержден.
- Статус запроса изменен на `accepted`.
- Баланс пользователя, подтвердившего запрос, уменьшается на сумму перевода.
- Баланс инициатора запроса увеличивается на сумму перевода.
