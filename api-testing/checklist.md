# Checklist

## Регистрация пользователя (doRegister)

### 1. Позитивные сценарии
- [ ] Регистрация с валидными данными
- [ ] Авторизация после успешной регистрации

---

### 2. Обязательные поля
- [ ] Пустой email
- [ ] Пустой name
- [ ] Пустой password

---

### 3. Валидация email
- [ ] Невалидный email (без @)
- [ ] Email с пробелами
- [ ] Дубликат email

---

### 4. Валидация password
- [ ] Короткий пароль (3 символа)
- [ ] Пароль из пробелов
- [ ] Пароль с пробелами по краям
- [ ] Длинный пароль (50 символов)
- [ ] Очень длинный пароль (255 символов)

---

### 5. Валидация name
- [ ] Name с пробелами по краям
- [ ] Name из пробелов
- [ ] Name с несколькими пробелами
- [ ] Спецсимволы в name

---

### 6. Уникальность
- [ ] Дубликат name
- [ ] Name в другом регистре
- [ ] Дубликат email

---

### 7. Консистентность
- [ ] Данные совпадают между API и GUI (email, name)
- [ ] Регистрация и авторизация (вход возможен после регистрации)

---

### 8. Обработка ошибок API
- [ ] HTTP статус соответствует результату (2xx / 4xx)
- [ ] Сообщение об ошибке корректное (без орфографических ошибок)
