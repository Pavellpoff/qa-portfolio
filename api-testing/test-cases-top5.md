# Test-cases-top5

## 1. Регистрация с пустым паролем

**Предусловие:**
- email = "test@example.com"
- name = "Pavel"
- password = ""

**Шаги:**
1. Отправить запрос на регистрацию
2. Убедиться, что пользователь создан
3. Попробовать авторизоваться с теми же данными

**Ожидаемый результат:**
- API возвращает ошибку валидации (HTTP 4xx)
- пользователь не создаётся
- авторизация невозможна

**Фактический результат:**
- API возвращает HTTP 200 ❌
- пользователь создаётся
- пароль сохраняется в виде хэша
- авторизация невозможна

**Вывод:**
Система допускает создание невалидных аккаунтов.
Нарушена логика: регистрация успешна, но вход невозможен.

## 2. Неконсистентная обработка пробелов в password

**Предусловие:**
- email = "test@example.com"

**Сценарий 1:**
password = "   "

**Сценарий 2:**
password = " r23 "

**Шаги:**
1. Зарегистрировать пользователя с каждым из паролей
2. Попробовать авторизоваться

**Ожидаемый результат:**
- пробелы обрабатываются единообразно:
  - либо обрезаются
  - либо значение считается невалидным

**Фактический результат:**
- "   " → пользователь создаётся, авторизация невозможна ❌
- " r23 " → пробелы обрезаются, авторизация успешна

**Вывод:**
Неконсистентная обработка входных данных.
Система по-разному обрабатывает пробелы в одном и том же поле.

## 3. API всегда возвращает HTTP 200 при ошибках

**Предусловие:**
- невалидные данные (например, пустой email)

**Шаги:**
1. Отправить запрос с невалидными данными

**Ожидаемый результат:**
- HTTP статус отражает ошибку (4xx)
- пользователь не создаётся

**Фактический результат:**
- HTTP статус = 200 ❌
- ошибка передаётся только в теле ответа

**Вывод:**
Нарушен API-контракт.
Клиент не может корректно определить результат запроса по HTTP статусу.

## 4. Неконсистентность данных между API и GUI (name с пробелами)

**Предусловие:**
- email = "test@example.com"
- name = " Александр   Петров "
- password = "123456"

**Шаги:**
1. Зарегистрировать пользователя
2. Проверить данные в API
3. Проверить данные в GUI

**Ожидаемый результат:**
- данные пользователя в API и GUI совпадают
- пробелы обрабатываются одинаково

**Фактический результат:**
- API: name = "Александр   Петров" (обрезаны края)
- GUI: name = " Александр   Петров " ❌

**Вывод:**
Несоответствие данных между API и GUI.
Система отображает и хранит разные значения.

## 5. Ошибка 500 при невалидных данных (спецсимволы в name)

**Предусловие:**
- email = "test@example.com"
- name = "t&*(+/"
- password = "123456"

**Шаги:**
1. Отправить запрос на регистрацию

**Ожидаемый результат:**
- HTTP статус 4xx
- сообщение об ошибке валидации
- пользователь не создаётся

**Фактический результат:**
- HTTP статус = 500 ❌
- сообщение: "Bad Request"
- пользователь не создан

**Вывод:**
Сервер некорректно обрабатывает невалидные данные.
Отсутствует обработка ошибок на стороне сервера.
