# 🧪 API Testing — Restful Booker

Проект - тестирование REST API сервиса **Restful Booker** с использованием Postman.

Тестируемый сервис: https://restful-booker.herokuapp.com

Документация: https://restful-booker.herokuapp.com/apidoc/index.html#api-Booking-UpdateBooking


## Описание проекта

В рамках проекта проведено функциональное тестирование API сервиса бронирования.
Проверены основные сценарии работы системы, включая авторизацию, создание, получение, обновление и фильтрацию бронирований.


## Цель

* Проверить корректность работы API
* Выявить дефекты в обработке запросов и данных
* Оценить соответствие API REST-практикам


## Используемые инструменты

* Postman
* GitHub


## Структура проекта


## Область тестирования

### Auth

* Получение токена
* Проверка валидной и невалидной авторизации

### Booking

* Получение списка бронирований
* Фильтрация (по имени, датам)
* Создание бронирования
* Обновление бронирования
* Проверка валидации данных


## Покрытие тестами

* Всего тест-кейсов: 18
* Позитивные сценарии
* Негативные сценарии
* Пограничные значения 


## Найденные дефекты

В ходе тестирования выявлены следующие проблемы:

* Некорректные HTTP-статусы (200/403 вместо 400/404)
* Ошибка 500 при невалидных параметрах
* Отсутствие валидации входных данных
* Сохранение некорректных значений дат (`0NaN-aN-aN`)
* Неконсистентная обработка параметров


## Результаты тестирования

* Пройдено тестов: 12
* Провалено тестов: 6

API выполняет основные функции, однако содержит критические проблемы в валидации и обработке ошибок.


## Вывод

Сервис реализует базовый функционал бронирования, но требует доработки:

* Валидации входных данных
* Обработки ошибок
* Соответствия REST-стандартам


## Как использовать

1. Импортировать коллекцию из папки `/postman` в Postman
2. Установить базовый URL:

```id="c7h2mz"

```

3. Запустить запросы и тесты
