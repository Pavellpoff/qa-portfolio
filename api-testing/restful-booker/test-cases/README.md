#Test Cases (Full)

## 1. CreateToken с валидными данными

### Метод: 
POST /auth

### Тип: 
Позитивный

### Предусловие:
API доступно

### Шаги:

Отправить POST запрос на /auth
В теле запроса передать:
{
  "username": "admin",
  "password": "password123"
}

Ожидаемый результат:

HTTP 200 OK
В ответе присутствует поле token
Тип значения token — string
Значение token не пустое

## 2. CreateToken с невалидными данными

Метод: POST /auth
Тип: Негативный

Предусловие:
API доступно

Шаги:

Отправить POST запрос на /auth
Передать невалидные данные:
неверный username
неверный password
пустые значения

Ожидаемый результат:

HTTP 400 Bad Request (при некорректных данных)
или HTTP 401 Unauthorized (при неверных кредах)
В ответе присутствует сообщение об ошибке
