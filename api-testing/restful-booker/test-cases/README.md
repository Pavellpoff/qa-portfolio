# API Test Cases - Restful Booker

Данный документ содержит тест-кейсы для проверки API сервиса Restful Booker.

**Покрытие:**

* Auth (CreateToken)
* Booking (GetBookingIds)
* Booking (GetBooking)
* Booking (CreateBooking)
* Booking (UpdateBooking)


**Инструменты:**

* Postman

---

## 1. CreateToken с валидными данными

**Метод:** POST /auth

**Тип:** Позитивный

**Предусловие:**
API доступно

**Шаги:**

1. Отправить POST запрос на `/auth`
2. В теле запроса передать:

```json
{
  "username": "admin",
  "password": "password123"
}
```

**Ожидаемый результат:**

* HTTP 200 OK
* В ответе присутствует поле `token`
* Тип значения `token` — string
* Значение `token` не пустое

---

## 2. CreateToken с невалидными данными

**Метод:** POST /auth

**Тип:** Негативный

**Предусловие:**
API доступно

**Шаги:**

1. Отправить POST запрос на `/auth`
2. Передать невалидные данные:

   * неверный username
   * неверный password
   * пустые значения

**Ожидаемый результат:**

* HTTP 400 Bad Request (при некорректных данных)
* или HTTP 401 Unauthorized (при неверных кредах)
* В ответе присутствует сообщение об ошибке

## 7. Получение списка booking ID по фильтру checkin/checkout

**Метод:** GET /booking

**Тип:** Позитивный

**Предусловие:**
API доступно
В системе существуют бронирования с заданными датами


### Шаги

1. Отправить GET запрос на
   `/booking?checkin=2016-07-05&checkout=2022-01-09`


### Ожидаемый результат

* HTTP 200 OK
* Тело ответа представляет собой массив
* Массив не пустой (если данные существуют)
* Каждый элемент массива — объект
* Каждый объект содержит поле `bookingid`
* Тип значения `bookingid` — number


### Пример ответа

```json id="e7h3vd"
[
  { "bookingid": 7 },
  { "bookingid": 11 }
]
```


### Примечание

Для полной проверки корректности фильтрации необходимо:

* выполнить GET `/booking/{id}` для каждого полученного `bookingid`
* убедиться, что даты `checkin` и `checkout` соответствуют заданным параметрам

## 8. Получение списка booking ID с некорректными значениями checkin/checkout

**Метод:** GET /booking

**Тип:** Негативный

**Предусловие:**
API доступно


### Шаги

1. Отправить GET запрос на `/booking` с некорректными параметрами:

   * `checkin="", checkout = 2022-01-09`
   * `checkin = 05.07.2019, checkout=""`


### Ожидаемый результат

* API корректно обрабатывает невалидные входные данные
* Возможные варианты поведения:

  * HTTP 400 Bad Request с описанием ошибки или HTTP 200 OK с пустым массивом
* API не должен возвращать ошибку (5xx)


### Примечание

Некорректные форматы дат и пустые значения должны валидироваться на стороне API.
