# Test Plan

## 1. Объект тестирования
API регистрации пользователей (метод doRegister)

---

## 2. Цель тестирования
Проверить корректность работы регистрации пользователей, включая:
- валидацию входных данных
- обработку граничных значений
- консистентность данных между API и GUI
- корректность бизнес-логики (уникальность, авторизация)

---

## 3. Область тестирования

### Включено:
- регистрация пользователя
- валидация полей:
  - email
  - name
  - password
- обработка:
  - пустых значений
  - некорректных значений
  - граничных значений (длина, пробелы)
- проверка уникальности:
  - email
  - name
- консистентность:
  - API / GUI
  - регистрация и авторизация

### Не включено:
- нагрузочное тестирование
- безопасность (глубокая)
- UI тестирование вне сценария регистрации

---

## 4. Подход к тестированию

Использованы следующие подходы:
- позитивное тестирование
- негативное тестирование
- граничные значения (boundary testing)
- анализ консистентности системы

---

## 5. Тестовые данные

Используются:
- валидные данные (корректный email, name, password)
- невалидные данные:
  - пустые значения
  - некорректный email
  - спецсимволы
- граничные значения:
  - короткий пароль
  - длинный пароль (50 и 255 символов)
  - строки с пробелами

---

## 6. Критерии успешности

- пользователь создается при валидных данных
- пользователь не создается при невалидных данных
- данные корректно сохраняются и отображаются
- возможна авторизация после регистрации
- API возвращает корректные статусы (2xx / 4xx)

---

## 7. Найденные риски

- API возвращает HTTP 200 при ошибках
- возможна регистрация с пустым паролем
- неконсистентная обработка пробелов
- различия между API и GUI
- отсутствие ограничений на длину пароля
