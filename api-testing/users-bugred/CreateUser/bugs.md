# 🐞 Bug Reports — CreateUser

---

## 🔴 BUG-01: API возвращает HTTP 200 при ошибках валидации

**Severity:** High

**Описание:**
API возвращает HTTP 200 даже в случаях, когда запрос содержит невалидные данные и в теле ответа присутствует сообщение об ошибке.

**Предусловие:**
Валидный SOAP-запрос CreateUser

**Шаги:**

1. Отправить запрос с несуществующим `tasks` (например: 5)
2. Получить ответ API

**Фактический результат:**

* HTTP статус: 200
* тело ответа содержит сообщение об ошибке: "задачи не найдены"

**Ожидаемый результат:**

* HTTP статус: 4xx
* сообщение об ошибке в теле ответа

**Влияние:**
Невозможно корректно обрабатывать ошибки на стороне клиента.

## 🔴 BUG-02: Отсутствует структурированный признак ошибки

**Severity:** Medium

**Описание:**
Ответ API не содержит поля `status`, `error code` или аналогичного признака ошибки. Ошибки передаются только текстом.

**Шаги:**

1. Отправить невалидный запрос
2. Проверить тело ответа

**Фактический результат:**

* только текстовое сообщение

**Ожидаемый результат:**

* наличие структурированного поля ошибки

**Влияние:**
Усложняет автоматизацию и обработку ошибок.

---

## 🔴 BUG-03: Пример в документации содержит невалидные данные

**Severity:** High

**Описание:**
Использование значений из документации (`companies = 36`) приводит к ошибке "Компании не найдены".

**Шаги:**

1. Отправить запрос с данными из документации
2. Проверить результат

**Фактический результат:**

* companies не применяются
* ошибка в GUI

**Ожидаемый результат:**

* пример должен быть полностью рабочим

**Влияние:**
Невозможно выполнить позитивный сценарий.

---

## 🔴 BUG-04: Частичное выполнение операции (нарушение атомарности)

**Severity:** Critical

**Описание:**
При передаче невалидного `companies` пользователь создаётся, но компании не привязываются.

**Фактический результат:**

* пользователь создан
* companies не применились

**Ожидаемый результат:**

* операция должна быть атомарной:

  * либо полностью успешной
  * либо полностью отклонённой

**Влияние:**
Нарушение целостности данных.

## 🔴 BUG-05: Смещение параметров при отсутствии tasks

**Severity:** Critical

**Описание:**
При отсутствии поля `tasks` значение из `companies` подставляется в `tasks`.

**Фактический результат:**

* tasks = значение companies

**Ожидаемый результат:**

* ошибка валидации

**Влияние:**
Искажение данных пользователя.

## 🔴 BUG-07: Некорректная интерпретация пустого значения

**Severity:** Medium

**Описание:**
Система не различает пустое значение и несуществующее значение.

**Шаги:**
Отправка запроса с пустым значением в массиве tasks:
<tasks soapenc:arrayType="xsd:int[1]">
    <item></item>
</tasks>

**Фактический результат:**

* сообщение: "задачи не найдены"

**Ожидаемый результат:**

* корректное сообщение (например: "пустое значение")

## 🔴 BUG 9: Некорректные значения по умолчанию

**Severity:** Medium

**Описание:**
При создании пользователя без передачи полей birthday и date_start, в GUI этим полям автоматически присваивается значение 0.

**Влияние**
 - может привести к ошибкам при дальнейшей обработке данных
 - нарушает консистентность между API и GUI
