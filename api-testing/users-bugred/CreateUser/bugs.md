# Bug Reports — CreateUser

---

## BUG 1: API возвращает HTTP 200 при ошибках валидации

**Серьезность:** Высокая

**Описание:**
При передаче невалидных данных API возвращает HTTP статус 200, несмотря на наличие ошибки в теле ответа.

**Шаги:**

1. Отправить запрос с невалидными tasks
2. Получить ответ

**Фактический результат:**

* HTTP 200
* сообщение об ошибке в теле ответа

**Ожидаемый результат:**

* HTTP 4xx
* ошибка в теле ответа

## BUG 2: Отсутствует структурированный признак ошибки в API

**Серьезность:** Средняя

**Описание:**
Ответ API не содержит поля `status` или `error code`, что затрудняет обработку ошибок на стороне клиента.

## 🔴 BUG-03: Пример в документации содержит невалидные данные

**Серьезность:** Высокая

**Описание:**
Использование значений `companies = 36` из документации приводит к ошибке "Компании не найдены".

**Ожидаемый результат:**
Примеры в документации должны быть валидными и воспроизводимыми.

---

## 🔴 BUG-04: Частичное выполнение операции (нарушение целостности)

**Серьезность:** Критическая

**Описание:**
При передаче невалидного `companies` пользователь создаётся, но компании не привязываются.

**Фактический результат:**

* пользователь создан
* companies не применились

**Ожидаемый результат:**

* либо операция полностью отклоняется
* либо возвращается ошибка

---

## 🔴 BUG-05: Смещение параметров при отсутствии tasks

**Серьезность:** Критическая

**Описание:**
При отсутствии поля `tasks` значение из `companies` подставляется в `tasks`.
