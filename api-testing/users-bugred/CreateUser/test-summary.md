# Отчет о методе CreateUser

---

## Общая информация

**Тестируемый метод:** CreateUser

**Тип API:** SOAP

**Цель тестирования:**

Проверка корректности:

* валидации данных
* обработки обязательных и необязательных полей
* обработки ошибок
* согласованности API и GUI

---

## Объем тестирования

Проведено тестирование следующих сценариев:

* невалидные значения обязательных полей
* отсутствие обязательных полей
* пустые значения
* данные из документации
* валидация необязательных полей

---

## Обнаруженные дефекты

В ходе тестирования выявлены дефекты различных уровней:

* Критические - частичное выполнение операций
* Высокие - некорректные HTTP статусы, ошибки документации
* Средние - проблемы валидации и сообщений
* Низкие - орфографические ошибки

---

## Ключевые проблемы

### 1. Неконсистентность системы

* расхождения между API, GUI и документацией

---

### 2. Некорректная обработка HTTP статусов

* API возвращает HTTP 200 при ошибках

---

### 3. Отсутствие структурированной обработки ошибок

* нет полей типа: `status` / `error code`

---

### 4. Нарушение целостности данных

* операции выполняются частично
* данные могут искажаться

---

### 5. Проблемы валидации

* отсутствует проверка обязательных полей
* система не различает типы ошибок

---

### 6. Проблемы документации

* отсутствуют валидные ID
* примеры содержат невалидные данные

---

### 7. Некорректная работа GUI

* отображение не соответствует API
* используются некорректные значения по умолчанию

---

## Ограничения тестирования

Из-за критических дефектов:

* невозможно выполнить полноценное позитивное тестирование
* дальнейшее тестирование ограничено

---

## Рекомендация

Метод не готов к использованию / релизу

---

## Итог

В ходе тестирования выявлены критические проблемы, влияющие на:

* корректность работы системы
* достоверность данных
* возможность дальнейшего тестирования

Метод требует доработки перед дальнейшим использованием.
