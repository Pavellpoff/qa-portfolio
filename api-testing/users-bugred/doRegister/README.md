## Тестирование метода doRegister

### Описание

Тестирование метода doRegister сервиса users.bugred.ru

---

### Цель

Проверка:

* валидации входных данных (email, name, password)
* обязательных полей
* обработки ошибок
* консистентности данных между API и GUI
* корректности бизнес-логики (регистрация → авторизация, уникальность)

---

### Что протестировано

* обязательные поля (email, name, password)
* невалидные значения:

  * некорректный email
  * спецсимволы в name
* пустые значения
* граничные значения:

  * короткий пароль
  * длинный пароль (50 и 255 символов)
  * значения с пробелами
* обработка дубликатов (email, name)
* консистентность:

  * API ↔ GUI
  * регистрация → авторизация

---

### Основные дефекты

* API всегда возвращает HTTP 200 при ошибках
* возможна регистрация с пустым паролем
* неконсистентная обработка пробелов
* различия данных между API и GUI
* ошибка 500 при невалидных данных
* регистрозависимость проверки уникальности name
* некорректные сообщения об ошибках (формат, орфография)

---

### Итог

Сервис содержит критические дефекты:

* HTTP статусы не отражают результат
* отсутствует единая логика валидации данных
* возможна регистрация некорректных пользователей
* обнаружена неконсистентность между компонентами системы
