# Test Cases (Full)

## 1. Позитивный сценарий

### 1.1 Регистрация с валидными данными
**Данные:**
- email: новый
- name: валидный
- password: валидный

**Ожидаемый результат:**
- пользователь создан
- email и name совпадают
- пароль сохраняется в виде хэша
- авторизация успешна

**Фактический результат:**
- HTTP статус = 200
- пользователь создан
- авторизация успешна

**Вывод:**
Система корректно обрабатывает валидные данные

---

## 2. Обязательные поля

### 2.1 Пустой email
**Ожидаемый результат:**
- ошибка валидации
- пользователь не создан

**Фактический результат:**
- HTTP 200 ❌
- сообщение содержит ошибку
- пользователь не создан

**Вывод:**
Некорректный HTTP статус + ошибка в тексте

---

### 2.2 Пустой name
**Ожидаемый результат:**
- ошибка
- пользователь не создан

**Фактический результат:**
- HTTP 200 ❌
- сообщение: "поле фио является обязательным"

**Вывод:**
- некорректный HTTP статус
- несоответствие терминологии (ФИО vs name)

---

### 2.3 Пустой пароль
**Ожидаемый результат:**
- ошибка
- пользователь не создан

**Фактический результат:**
- HTTP 200 ❌
- пользователь создан
- авторизация невозможна

**Вывод:**
Создаются невалидные аккаунты (критический баг)
