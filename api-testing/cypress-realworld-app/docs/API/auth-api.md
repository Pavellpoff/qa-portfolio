# Авторизация (API)

## Описание

Набор API-автотестов проверяет работу эндпоинта авторизации пользователя.

---

## Покрываемые сценарии

| № | Сценарий | Статус |
|---|----------|:------:|
| 1 | Успешная авторизация | ✅ |
| 2 | Авторизация с пустыми username и password | ✅ |
| 3 | Авторизация с неверным паролем | ✅ |
| 4 | Авторизация несуществующего пользователя | ✅ |
| 5 | Авторизация с пустым username | ❌ |
| 6 | Авторизация username отсутствует вообще, нет поля | ❌ |
| 7 | Авторизация с username и password null | ❌ |
| 8 | Авторизация с SQL Injection| ❌ |


---

## Проверки

### 1. Успешная авторизация

Проверяется:

- получение HTTP 200;
- наличие объекта пользователя;
- корректность имени пользователя;
- наличие идентификатора пользователя;
- наличие Email;
- наличие баланса пользователя.

---

### 2. Авторизация с пустыми username и password

Проверяется:

- получение HTTP 400;
- корректное сообщение об ошибке.

---

### 3. Авторизация с неверным паролем

Проверяется:

- отправка запроса с существующим username и неверным password;
- получение HTTP 401;
- сообщение Unauthorized.

---

### 4. Авторизация с несуществующим пользователем

Проверяется:

- отправка запроса с несуществующим username;
- получение HTTP 401;
- сообщение Unauthorized.
