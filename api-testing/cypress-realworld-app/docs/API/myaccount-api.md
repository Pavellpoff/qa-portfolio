# Личный кабинет (API)

## Описание

Данный набор API-автотестов проверяет работу эндпоинтов управления профилем пользователя в приложении Cypress Real World App.

Основное внимание уделено проверке изменения данных пользователя, сохранению изменений, обработке некорректных входных данных и проверке ограничений доступа.

## Проверки

### 1. Изменение профиля пользователя

Проверяется:

* успешное выполнение PATCH-запроса;
* получение HTTP 204;
* корректное сохранение измененных данных;
* получение обновленных данных через GET-запрос.

---

### 2. Проверка обязательных полей

Проверяется возможность сохранения профиля при пустых значениях:

* First Name;
* Last Name;
* Email;
* Phone Number.

После изменения выполняется GET-запрос для подтверждения фактического сохранения данных.

---

### 3. Проверка некорректных данных

Проверяется сохранение профиля при передаче:

* Email в некорректном формате;
* Phone Number с недопустимой длиной.

После изменения выполняется проверка сохраненных данных.

---

### 4. Проверка ограничений доступа

Проверяется попытка изменения профиля другого пользователя посредством PATCH-запроса.

Тест предназначен для исследования поведения API при обращении к ресурсу другого пользователя.

---

### 5. Проверка доступа без авторизации

Проверяется:

* попытка выполнения `PATCH`-запроса без авторизации;
* получение HTTP `401 Unauthorized`;
* получение сообщения об ошибке `Unauthorized`.

---
### 6. Проверка обработки PATCH-запроса без body

Проверяется:

* выполнение PATCH-запроса без body;
* получение HTTP `422 Unprocessable Entity`;
* получение информации об ошибке валидации;
* отсутствие изменений в данных пользователя.

---

## Проверяемые HTTP-методы

| Метод | Назначение                          |
| ----- | ----------------------------------- |
| POST  | Авторизация пользователя            |
| PATCH | Изменение данных профиля            |
| GET   | Получение информации о пользователе |

---

## Проверяемые HTTP-статусы

| Код | Значение                             |
| --- | ------------------------------------ |
| 200 | Данные пользователя успешно получены |
| 204 | Данные пользователя успешно изменены |

---

## Особенности реализации

В каждом тесте после выполнения PATCH-запроса выполняется дополнительный GET-запрос.

Такой подход позволяет подтвердить не только успешное выполнение запроса, но и фактическое сохранение изменений в системе.
