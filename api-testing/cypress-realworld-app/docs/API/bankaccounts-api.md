# Банковские счета (API)

## Описание

Данный набор API-автотестов проверяет создание и удаление банковских счетов, а также исследует серверную валидацию входных данных.

Основное внимание уделено проверке обработки невалидных значений и сохранению данных банковского счета.

---

## Проверки

### 1. Создание банковского счета

Проверяется:

* успешное выполнение GraphQL-мутации `CreateBankAccount`;
* получение HTTP 200;
* отсутствие ошибок GraphQL;
* корректность сохраненных данных;
* успешное выполнение GraphQL-мутации `DeleteBankAccount`;
* успешное удаление созданного банковского счета.

---

### 2. Проверка обязательных полей

Проверяется создание банковского счета при передаче пустых значений:

* Bank Name;
* Routing Number;
* Account Number.

---

### 3. Проверка отсутствия серверной валидации

Проверяется создание банковского счета при передаче:

* чрезмерно длинных значений `Account Number`;
* чрезмерно длинных значений `Routing Number`;
* буквенных символов;
* специальных символов.

---

## Проверяемые GraphQL-операции

| Операция            | Назначение                 |
| ------------------- | -------------------------- |
| `CreateBankAccount` | Создание банковского счета |
| `DeleteBankAccount` | Удаление банковского счета |

---

## Проверяемые HTTP-статусы

| Код | Значение                        |
| --- | ------------------------------- |
| 200 | GraphQL-запрос успешно выполнен |

---

## Особенности реализации

После создания банковского счета выполняется проверка данных, возвращаемых GraphQL API.

В позитивном сценарии дополнительно выполняется удаление созданного банковского счета для проверки корректной работы операции удаления.

Негативные сценарии используются для исследования серверной валидации и проверки обработки некорректных входных данных.
