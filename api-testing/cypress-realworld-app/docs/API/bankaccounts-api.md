# Банковские счета (API)

## Описание

Данный набор API-автотестов проверяет создание и удаление банковских счетов, а также исследует серверную валидацию входных данных.

Основное внимание уделено проверке обработки невалидных значений и сохранению данных банковского счета.

---

## Покрываемые сценарии

| № | Сценарий                                                           | Статус |
| - | ------------------------------------------------------------------ | :----: |
| 1 | Создание банковского счёта                                         |    ✅   |
| 2 | Удаление банковского счёта                                         |    ✅   |
| 3 | Создание счёта с пустым названием банка                            |    ✅   |
| 4 | Создание счёта с пустым Routing Number                             |    ✅   |
| 5 | Создание счёта с пустым Account Number                             |    ✅   |
| 6 | Создание счёта с длинными номерами счёта и маршрутизации           |    ✅   |
| 7 | Создание счёта с буквами и специальными символами в числовых полях |    ✅   |

---

## Проверки

### 1. Создание банковского счета

Проверяется:

* успешное выполнение GraphQL-мутации `CreateBankAccount`;
* получение HTTP 200;
* корректность сохранённых данных;
* успешное удаление созданного банковского счета посредством мутации `DeleteBankAccount`.

---

### 2. Проверка обязательных полей

Проверяется создание банковского счета при передаче пустых значений:

* Bank Name;
* Routing Number;
* Account Number.

---

### 3. Проверка отсутствия серверной валидации

Проверяется создание банковского счёта при передаче:

* чрезмерно длинных значений `Account Number`;
* чрезмерно длинных значений `Routing Number`;
* буквенных символов;
* специальных символов.

---

## Проверяемые GraphQL-операции

| Операция            | Назначение                 |
| ------------------- | -------------------------- |
| `CreateBankAccount` | Создание банковского счёта |
| `DeleteBankAccount` | Удаление банковского счёта |

---

## Проверяемые HTTP-статусы

| Код | Значение                        |
| --- | ------------------------------- |
| 200 | GraphQL-запрос успешно выполнен |

---

## Особенности реализации

После создания банковского счёта выполняется проверка данных, возвращаемых GraphQL API.

В позитивном сценарии дополнительно выполняется удаление созданного банковского счёта для проверки корректной работы операции удаления.

Негативные сценарии используются для исследования серверной валидации и проверки обработки некорректных входных данных.
