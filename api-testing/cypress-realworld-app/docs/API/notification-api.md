# Уведомления (API)

## Описание

Данный набор API-автотестов проверяет работу эндпоинтов управления уведомлениями пользователя.

Основное внимание уделено проверке изменения статуса уведомления и корректному обновлению списка уведомлений после выполнения операции.

---

## Покрываемые сценарии

| № | Сценарий                             | Статус |
| - | ------------------------------------ | :----: |
| 1 | Пометка уведомления как прочитанного | ✅ |
| 2 | PATCH с несуществующим notificationId (исследование поведения API) | ✅ |


---

## Проверки

### 1. Пометка уведомления как прочитанного

Проверяется:

* успешная авторизация пользователя;
* получение списка уведомлений;
* успешное изменение статуса уведомления через PATCH-запрос;
* получение HTTP 204;
* повторное получение списка уведомлений;
* отсутствие обработанного уведомления в списке непрочитанных уведомлений.


### 2. Исследование обработки несуществующего notificationId

Проверяется:

* выполнение PATCH-запроса с несуществующим `notificationId`;
* анализ фактического поведения API;
* получение HTTP `204 No Content`.


---

## Проверяемые HTTP-методы

| Метод | Назначение                    |
| ----- | ----------------------------- |
| POST  | Авторизация пользователя      |
| GET   | Получение списка уведомлений  |
| PATCH | Изменение статуса уведомления |

---

## Проверяемые HTTP-статусы

| Код | Значение                      |
| --- | ----------------------------- |
| 200 | Запрос выполнен успешно       |
| 204 | Уведомление успешно обновлено |

---

## Особенности реализации

После изменения статуса уведомления выполняется повторный запрос на получение списка уведомлений.

Тест подтверждает не только успешное выполнение PATCH-запроса, но и фактическое изменение состояния системы: прочитанное уведомление больше не возвращается в списке непрочитанных уведомлений.
