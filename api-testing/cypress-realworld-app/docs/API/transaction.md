# Денежные переводы (API)

## Описание

Данный набор API-автотестов проверяет работу эндпоинта создания запросов на перевод денежных средств между пользователями.

Основное внимание уделено проверке бизнес-логики создания запросов на перевод, обработке некорректных входных данных, исследованию граничных значений и анализу поведения API в нестандартных сценариях.

## Проверки

### 1. Успешное создание нового запроса на перевод 

Проверяется:

* успешная авторизация пользователя;
* получение идентификатора отправителя;
* получение идентификатора получателя;
* успешное создание нового запроса на перевод;
* корректность данных созданного запроса на перевод.

---

### 2. Проверка обязательных полей

Проверяется обработка следующих сценариев:

* пустое значение `amount`;
* пустое значение `description`.

---

### 3. Проверка граничных и нестандартных значений

Проверяется обработка:

* отрицательных значений суммы;
* очень больших значений суммы;
* дробных значений суммы;
* строки, состоящей только из пробелов, в поле `description`.

---

### 4. Проверка бизнес-логики

Проверяется:

* создание запроса на перевод несуществующему пользователю;
* попытка создания запроса на перевод самому себе.

---

## Проверяемые HTTP-методы

| Метод | Назначение                              |
| ----- | ----------------------------------------|
| POST  | Авторизация пользователя                |
| GET   | Получение списка пользователей          |
| POST  | Создание нового запроса на перевод      |
| GET   | Получение списка транзакций             |

---

## Проверяемые HTTP-статусы

| Код | Значение                                               |
| --- | ------------------------------------------------------ |
| 200 | Запрос успешно выполнен                                |
| 422 | Ошибка валидации входных данных                        |
| 500 | Внутренняя ошибка сервера (исследовательский сценарий) |

---

## Особенности реализации

Автотесты не ограничиваются проверкой HTTP-статусов и структуры ответа API.

Дополнительно исследуется поведение системы при передаче некорректных данных, анализируется обработка граничных значений, проверяется соблюдение бизнес-правил, а также выявляются различия между ожидаемым и фактическим поведением API.

Несколько тестов используются для подтверждения потенциальных дефектов серверной логики и последующего оформления баг-репортов.
