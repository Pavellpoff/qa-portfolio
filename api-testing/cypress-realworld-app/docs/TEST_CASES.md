# Тест-кейсы

## TC 1. AUTH. Успешная авторизация пользователя

**Модуль:** Авторизация

**Предусловия**

- Пользователь зарегистрирован.
- Страница `/signin` открыта.

**Шаги**

1. Ввести корректный Username.
2. Ввести корректный Password.
3. Нажать кнопку **Sign In**.

**Ожидаемый результат**

- Пользователь успешно авторизован.
- Отображается главная страница приложения.
- Отображается имя пользователя.
- Сервер возвращает HTTP 200.

---

## TC 2.AUTH. Проверка обязательности полей авторизации

**Модуль:** Авторизация

**Предусловия**

- Страница `/signin` открыта.

**Шаги**

1. Не заполнять поля Username и Password.
2. Перевести фокус между полями.

**Ожидаемый результат**

- Отображается сообщение **Username is required**.
- Отображается сообщение **Password is required**.
- Авторизация недоступна.

---

## TC 3. AUTH. Авторизация с неверным паролем

**Модуль:** Авторизация

**Предусловия**

- Существует пользователь с корректным логином.

**Шаги**

1. Ввести корректный Username.
2. Ввести неверный Password.
3. Нажать **Sign In**.

**Ожидаемый результат**

- Авторизация не выполняется.
- Сервер возвращает HTTP 401.
- Отображается сообщение **Username or password is invalid**.

# 📝 Тест-кейсы — Личный кабинет

---

## TC ACCOUNT-1. Проверка обязательности полей профиля

**Модуль:** Личный кабинет

**Предусловия**

- Пользователь авторизован.
- Открыта страница **User Settings**.

**Шаги**

1. Очистить поля:
   - First Name;
   - Last Name;
   - Email;
   - Phone Number.
2. Проверить состояние кнопки **Save**.

**Ожидаемый результат**

- Кнопка сохранения недоступна.
- Отображаются сообщения валидации:
  - `Enter a first name`;
  - `Enter a last name`;
  - `Enter an email address`;
  - `Enter a phone number`.

---

## TC ACCOUNT-2. Проверка валидации Email и номера телефона

**Модуль:** Личный кабинет

**Предусловия**

- Пользователь авторизован.
- Открыта страница **User Settings**.

**Шаги**

1. Очистить поля **Email** и **Phone Number**.
2. Ввести некорректный Email.
3. Ввести номер телефона короче допустимой длины.
4. Проверить состояние формы.

**Ожидаемый результат**

- Кнопка сохранения недоступна.
- Отображается сообщение:
  - `Must contain a valid email address`;
  - `Phone number is not valid`.

---

## TC ACCOUNT-3. Проверка сохранения нестандартных данных в полях профиля

**Модуль:** Личный кабинет

**Предусловия**

- Пользователь авторизован.
- Открыта страница **User Settings**.

**Шаги**

1. Очистить поля **First Name**, **Last Name** и **Phone Number**.
2. Ввести в поле **First Name** числовое значение.
3. Ввести в поле **Last Name** длинную строку, содержащую буквы, цифры и специальные символы.
4. Ввести длинный номер телефона.
5. Сохранить изменения.

**Ожидаемый результат**

- Изменения успешно сохраняются.
- Новые значения отображаются в профиле пользователя.

> **Примечание:** Тест является исследовательским и предназначен для проверки поведения приложения при вводе нестандартных данных.
