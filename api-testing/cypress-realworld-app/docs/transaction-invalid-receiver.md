# Root Cause Analysis

## Симптом

GET /transactions возвращает HTTP 500.

## Что предшествовало

Создание транзакции с несуществующим receiverId.

## Что удалось установить

По стеку ошибки исключение возникает в функции

formatTransactionForApiResponse()

Во время формирования JSON сервер обращается к объекту receiver.

Пользователь отсутствует, то:

receiver == undefined

При обращении к receiver.avatar возникает 
TypeError

↓

GET /transactions

↓

500 Internal Server Error

## Предполагаемая причина

Отсутствует проверка существования получателя при создании транзакции.

Из-за этого в системе появляются неконсистентные данные.

При последующей обработке транзакции сервер не может сформировать корректный ответ.
