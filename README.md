# QA Portfolio

## О себе

Начинающий QA Engineer. Практикую тестирование API и изучаю процессы тестирования.

## Навыки

* API testing (SOAP / REST)
* Test cases, чек-листы
* Bug reporting
* Валидация данных и граничные значения

## Проект: API тестирование регистрации пользователей

Провёл тестирование API сервиса регистрации пользователей (метод doRegister).

### Что проверял:

* Валидацию полей (email, name, password)
* Обработку граничных значений
* Консистентность API и GUI
* Связку регистрация → авторизация

### Найденные баги:

* API всегда возвращает HTTP 200
* Возможна регистрация с пустым паролем
* Неконсистентная обработка пробелов
* Регистрозависимость уникальности name
