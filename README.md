# QA Portfolio

## 👨‍💻 О себе

Начинающий QA Engineer с практическим опытом тестирования API.
В рамках проекта протестировал сервис регистрации пользователей, включая валидацию данных, бизнес-логику и консистентность системы.

---

## 🛠 Навыки

* API testing (REST / SOAP)
* Написание тест-кейсов и чек-листов
* Bug reporting
* Тестирование валидации данных
* Граничные значения
* Анализ консистентности (API / GUI)

---

## 📌 Проект: Тестирование API регистрации пользователей
Тестируемый сервис
API: http://users.bugred.ru/

Документация: https://testbase.atlassian.net/wiki/spaces/USERS/pages/592674928/doRegister
Тип API: REST / SOAP

**Метод:** `doRegister`
**Инструменты:** SoapUI

### 🔍 Что проверял:

* Валидацию полей: email, name, password
* Обработку:

  * пустых значений
  * невалидных данных
  * граничных значений (длина, пробелы)
* Уникальность пользователей (email, name)
* Консистентность:

  * API / GUI
  * регистрация / авторизация

## 🐞 Найденные баги

* API всегда возвращает HTTP 200 при ошибках
* Возможна регистрация с пустым паролем
* Неконсистентная обработка пробелов
* Неконсистентность данных между API и GUI
* Ошибка 500 при невалидных данных
* Регистрозависимость уникальности name

---

## ⚠️ Выводы по системе

* HTTP статусы не отражают результат
* Отсутствует единая стратегия валидации данных
* Есть расхождения между API и GUI
* Система допускает создание некорректных пользователей
