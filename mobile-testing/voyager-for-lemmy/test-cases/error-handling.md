# Обработка серверных ошибок

## TC 7. Проверка обработки ошибки 500

**Окружение:**  
- Android Studio  
- API 33 “Tiramisu”, Android 13  
- Resolution: 2400×1800  
- 480 dpi  
- 6,4”  
- Proxyman  

**Шаги:**  
1. Перехватить response:  
   `GET /api/v3/post?id=...`  
2. Изменить ответ сервера на HTTP 500  

**Ожидаемый результат:**  
- приложение не падает  
- интерфейс не зависает  
- экран отображается корректно  

**Фактический результат:**  
- приложение продолжает отображать ранее загруженные данные поста  
- сообщение об ошибке не отображается  
- приложение не падает  

**Вывод:**  
Приложение устойчиво к HTTP 500 и продолжает отображать ранее загруженные данные поста.

## TC 8. Обработка ошибки 500 и пустого body

**Окружение:**  
- Android Studio  
- API 33 “Tiramisu”, Android 13  
- Resolution: 2400×1800  
- 480 dpi  
- 6,4”  
- Proxyman  

**Шаги:**  
1. Очистить кэш приложения  
2. Перезагрузить пост  
3. Перехватить response:  
   `GET /api/v3/post?id=...`  
4. Изменить ответ сервера на HTTP 500  
5. Заменить `response body` на пустой JSON: `{}`  

**Ожидаемый результат:**  
- приложение корректно обрабатывает ошибку сервера  
- приложение не падает  
- интерфейс не зависает и отображается корректно  

**Фактический результат:**  
- при HTTP 500 и пустом `response body` приложение продолжает отображать ранее загруженные данные поста  
- предположительно используется локальное кэширование данных  
- сообщение об ошибке не отображается  
- признаков падения приложения или нарушения интерфейса не обнаружено  

**Вывод:**  
Приложение устойчиво к ошибкам сервера, предположительно использует локальное кэширование данных и сохраняет usability.

## TC 9. Проверка обработки старых дат

**Окружение:**  
- Android Studio  
- API 33 “Tiramisu”, Android 13  
- Resolution: 2400×1800  
- 480 dpi  
- 6,4”  
- Proxyman  

**Шаги:**  
1. Очистить кэш приложения  
2. Перезагрузить пост  
3. Перехватить response:  
   `GET /api/v3/post?id=...`  

**Проверяемые сценарии:**  
- заменить значение поля `published` на `2025-05-14T14:27:26`  
- заменить значение поля `published` на `1900-05-14T14:27:26`  

**Ожидаемый результат:**  
- приложение не падает  
- интерфейс не зависает  
- экран отображается корректно  
- дата публикации отображается корректно в соответствии с измененным значением поля `published`  

**Фактический результат:**  
- дата поста отображается как `1 год назад` и `126 лет назад` соответственно, что соответствует измененным значениям поля `published`  
- признаков падения приложения или нарушения интерфейса не обнаружено  

**Вывод:**  
Приложение корректно обрабатывает и отображает измененные значения даты публикации, включая экстремально старые даты.
