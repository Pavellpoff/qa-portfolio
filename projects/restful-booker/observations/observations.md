## OBS 1. Некорректная интерпретация значения 0 в полях дат


### Описание

При передаче значения `0` в параметры `checkin` и `checkout` API интерпретирует его как дату `2000-01-01`.


### Шаги наблюдения:

1. Отправить POST запрос на `/booking`
2. В теле запроса указать:

```json id="b7k3zp"
{
  "bookingdates": {
    "checkin": 0,
    "checkout": 0
  }
}
```


### Фактический результат

* HTTP 200 OK
* Значения полей:

```json id="m4q8vd"
"checkin": "2000-01-01",
"checkout": "2000-01-01"
```


### Вывод

При передаче значения `0` API преобразует его в дату `2000-01-01`.
Такое поведение не является очевидным и не задокументировано.


### Требуется уточнение:

* Является ли это ожидаемым поведением
